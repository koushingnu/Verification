# 本人確認書類 画像検索システム

さくらサーバー上に保存された本人確認画像（別サイトのフォーム送信で配置されたもの）を、
社内から検索・プレビュー・ダウンロード保存するための簡易的な社内向けWebアプリです。

Next.js (App Router) 製で、Vercel上での稼働を想定しています。DBは使用せず、FTP(S)接続で
さくらサーバー上のファイルを直接検索します。

## 主な機能

- 登録済みのさくらサーバー（最大10台まで拡張可）から対象を選択
- ファイル名・パスに対するキーワード部分一致検索（再帰的にフォルダを走査）
- 検索結果のサムネイル表示・拡大プレビュー
- 画像のダウンロード（ブラウザ経由の保存。Vercelはサーバーレスのため、アプリ側にファイルは保持しません）
- アプリ全体にBasic認証を必須で適用（社内限定アクセス）

## セキュリティ上の設計方針

本人確認書類という機微な個人情報を扱うため、以下を前提に実装しています。

1. **Basic認証必須（フェイルクローズ）**
   `BASIC_AUTH_USER` / `BASIC_AUTH_PASSWORD` が未設定の場合、全リクエストを500エラーで拒否します。
   デフォルトで公開されてしまうことを防ぐためです。
2. **パストラバーサル対策**
   ユーザーが指定するパス（サブフォルダ・検索結果のファイルパス）は、必ずサーバーごとに固定された
   `BASE_DIR` 配下に正規化してから使用します。`../` 等でベースディレクトリの外に出ることはできません。
3. **接続先サーバーの固定**
   フロントエンドから任意のホストを指定することはできません。環境変数で事前登録した
   サーバーのみ、IDで選択します（SSRF対策）。
4. **拡張子ホワイトリスト**
   画像拡張子（jpg/jpeg/png/gif/webp/bmp/tif/tiff/heic/heif）以外のファイルは検索結果にも
   画像配信APIにも出しません。
5. **認証情報の非公開**
   FTP接続情報はサーバーサイド（Route Handler）のみで使用し、ブラウザには一切送信しません。
   `.env.local` はGit管理対象外です。
6. **通信の暗号化（推奨）**
   既定でFTPS（明示的TLS）を使用します。さくら側がFTPSに対応していない場合のみ
   `SAKURA_SERVER_n_SECURE=false` にしてください（平文通信になるためリスクを理解の上で設定してください）。

## セットアップ

### 1. 依存パッケージのインストール

```bash
npm install
```

### 2. 環境変数の設定

`.env.local.example` を `.env.local` にコピーし、内容を編集してください。

```bash
cp .env.local.example .env.local
```

- `BASIC_AUTH_USER` / `BASIC_AUTH_PASSWORD`: アプリ全体を保護する社内共通のBasic認証情報
- `SAKURA_SERVER_1_*` 〜 `SAKURA_SERVER_3_*`: 接続先のさくらサーバー情報
  - `HOST` が空のサーバーはサーバー一覧に表示されません
  - `BASE_DIR` はこのアプリから検索・閲覧できる範囲を制限するルートディレクトリです
  - `SECURE` は既定で `true`（FTPS）。プレーンFTPしか使えない場合のみ `false` にしてください

さらにサーバーを追加したい場合は `SAKURA_SERVER_4_*`、`SAKURA_SERVER_5_*` … と番号を増やすだけで
（最大10台まで）自動的に認識されます。

### 3. ローカルでの起動

```bash
npm run dev
```

`http://localhost:3000` を開き、Basic認証のダイアログで設定したユーザー名/パスワードを入力してください。

## Vercelへのデプロイ

1. GitHub等にリポジトリをpushし、Vercelでプロジェクトをインポートします。
2. Vercelの **Project Settings > Environment Variables** に `.env.local` と同じ内容を設定します
   （`.env.local` はGitにコミットされないため、必ず手動で設定してください）。
3. デプロイ後、割り当てられたURLにBasic認証情報でアクセスします。

### Vercel特有の制約について

- **実行時間の上限**: サーバーレス関数には実行時間の上限があります（Hobbyプランは最大60秒、
  Proプラン以上はより長く設定可能）。検索・画像取得のAPIには `maxDuration = 60` を設定していますが、
  対象フォルダのファイル数が非常に多い場合はタイムアウトする可能性があります。
  その場合は検索画面の「詳細検索オプション」からサブフォルダを指定し、走査範囲を絞り込んでください。
  併せて `SEARCH_MAX_DEPTH` / `SEARCH_MAX_RESULTS` / `SEARCH_MAX_SCANNED` で上限を調整できます。
- **永続ストレージなし**: Vercel上にファイルを保存する仕組みは持たせていません。「保存」操作は
  ブラウザ経由でのファイルダウンロードとして実装しています。

## ディレクトリ構成（主要ファイル）

```
src/
  proxy.ts                    # Basic認証（全ルート対象。旧middleware.ts、Next.js 16での新名称）
  lib/
    servers.ts                # 環境変数からサーバー一覧を構築
    pathSafety.ts              # パストラバーサル対策・拡張子チェック
    ftp.ts                     # FTPS接続・再帰検索・ファイルストリーム取得
    format.ts                  # 表示用フォーマット関数
  app/
    page.tsx                   # 検索UI（サーバー選択・キーワード検索・プレビュー・保存）
    api/servers/route.ts       # サーバー一覧API
    api/search/route.ts        # キーワード検索API
    api/image/route.ts         # 画像プレビュー/ダウンロードAPI
```

## 既知の制限・今後の改善候補

- ファイル名/パスの部分一致検索のみ対応（全文検索やOCRなどは非対応）
- 大量ファイルがあるフォルダでは再帰検索が遅くなるため、サブフォルダ指定を推奨
- 現状は共通Basic認証のみ。利用者ごとのアカウント管理や閲覧ログ（誰がいつどの画像を見たか）が
  必要な場合は別途認証基盤・監査ログの追加を検討してください
